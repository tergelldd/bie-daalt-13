# Self-Evaluation

## 1. Хэрэв шалгалт өнөөдөр болбол би энэ кодыг өөрөө бичиж чадах уу?

**Чадах хэсгүүд:**
- NestJS-ийн module/controller/service бүтэц, dependency injection-г ойлгосон тул scaffold хийж чадна
- `url-validation.ts`-ийн логик (`new URL()` parse, regex шалгалт) ойлгомжтой болсон
- Jest тестийн `describe`/`it`/`mock` бүтцийг бичиж чадна
- Prisma schema (`model ShortUrl { ... }`) болон migrate командыг мэднэ
- `$transaction`-ийн зорилго болон ашиглах аргыг тайлбарлаж чадна

**Бэрхшээлтэй хэсгүүд:**
- `PrismaPg` adapter-ийн initialization (`Pool` → `PrismaPg` → `super({ adapter })`) дарааллыг баримт бичгийгүй бичих бол эргэлзэнэ
- NestJS `@SkipThrottle`, `ThrottlerModule` тохиргооны нарийн ширийн зүйлсийг цээжилсэнгүй

## 2. Дахин хийнэ гэвэл юуг өөрөөр хийх вэ?

**Prisma хувилбарыг эхэндээ шалгах:**
`package.json`-д суусан Prisma-ийн version-г эхлээд шалгаж, тухайн version-ийн breaking change-уудыг official changelog-оос уншина. AI-д асуухаас өмнө баримт бичгийг харна.

**Interface эсвэл contract файл эхэндээ тогтоох:**
Controller болон Service-ийн функцийн нэрүүд зөрсөн алдаа давтагдахаас сэргийлэхийн тулд `app.service.interface.ts` файл үүсгэж, бүх функцийн нэр, параметр, return type-г тогтооно. Дараа нь AI-д "энэ interface-г хэрэгжүүлэх service бич" гэж тодорхой даалгавар өгнө.

**AI session-уудыг өдөр болгон хадгалах:**
Session log хожимдоо хадгалснаас болж зарим чухал ярилцлагыг мартсан. Цаашид chat дуусаагүй байхад товч log бичих дадалтай болно.

**Тест эхэндээ бичих (TDD):**
Тестийг код бичсэний дараа бичсэн нь зарим тохиолдолд тест нь кодын алдааг барьж чадалгүй "pass болж" байсан. Эхлээд failing test бичиж, дараа нь код бичвэл AI-ийн гаргасан кодыг илүү сайн шалгах боломжтой.

## 3. Энэ туршлагаас юу сурсан бэ?

**"Verify, don't trust" — практик утгаараа:**
AI-ийн санал болгосон код compile болж байна гэдэг нь зөв гэсэн үг биш. `url-validation.ts` import хийгдэхгүй байсан, redirect буруу төрөл буцааж байсан зэрэг алдаанууд runtime-д л илэрсэн. Кодыг ажиллуулж, тест бичиж, шалгах шатыг орхих боломжгүй.

**AI нь tool, expert биш:**
Prisma 7 шиг шинэ major version-ийн API-г AI мэдэхгүй байж болно. Training data cutoff-ийн дараа гарсан breaking change-уудыг AI буруу санал болгодог. Official баримт бичиг нь үргэлж эцсийн шүүгч.

**AI-тай хамтран ажиллах оновчтой арга:**
AI нь boilerplate, тест бүтэц, баримт бичиг зэрэг давтагддаг ажлуудад маш хурдан. Харин шинэ API, security-critical хэсэг, архитектурын шийдвэрт AI-ийн саналыг шалгаж баталгаажуулах хэрэгтэй. "AI бичсэн" нь "би ойлгосон" гэсэн үг биш — тайлбарлаж чадаагүй код нь өөрийнх биш.
